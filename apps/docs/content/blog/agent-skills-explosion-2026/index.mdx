Ten weeks ago, I didn't know what "agent skills" were. Now they're in my daily workflow, and apparently the same is true for hundreds of thousands of other developers — because the GitHub numbers are unlike anything I've seen this year.

Let me show you what I mean before I explain it.

It's a builder movement with receipts.

## What Even Is a "Skill"?

A quick explainer, because the naming is still messy.

Coding agents like Claude Code, Codex CLI, OpenCode, and others execute instructions from your project files. An "agent skill" is a small piece of instruction — usually markdown, sometimes a script — that tells the agent how to do something specific.

A skill might say: *"When I ask you to write a blog post, read these instructions first."* Or: *"Before touching any file in `/src/auth/`, check the security guidelines."* Or more practically: *"Use `trash` instead of `rm` when deleting files."*

The insight behind `affaan-m/everything-claude-code` (the 100K-star repo) is that you can dramatically change how an AI agent behaves by curating a library of these small instructions. The repo is half philosophy (how to structure your AGENTS.md), half practical toolkit (what skills are worth having).

It's closer to dotfiles culture than to traditional software libraries. You're not installing a dependency — you're shaping an agent's personality.

### Three kinds of "skills"

The naming is messy, but what people call "skills" today roughly falls into three distinct groups:

1. **Prompt-based reusable instructions** — Markdown files that set behavioral guardrails, coding style, or domain knowledge. Think `AGENTS.md`, `CLAUDE.md`, personal dotfiles-for-agents. This is the largest category by repo count.

2. **Tool wrappers** — Skills that give agents access to external capabilities, often aligned with MCP (Model Context Protocol). A skill might wrap a database query, an API call, or a browser action into something an agent can invoke by name.

3. **Workflow-level compositions** — Multi-step orchestration skills that chain together tools, memory, and sub-agents inside frameworks like OpenClaw, LangGraph, or CrewAI. These are the most complex and the least portable.

The distinction matters because they have very different shelf lives. Category 1 is agent-agnostic and will survive format wars. Category 3 is deeply framework-coupled and might not.

Forks on a dotfiles-style repo aren't people starting derivative projects. They're people taking the baseline and personalizing it. 13K forks on a 10-week-old repo means 13,000 developers have already created their own customized version. That's the real adoption signal.

## The Personal-Public Blend

Something unusual is happening in this category that I didn't expect: personal collections are performing as well as org-backed projects.

[mattpocock/skills](https://ossinsight.io/analyze/mattpocock/skills) (Matt Pocock's personal skills, 9,442 stars), [JimLiu/baoyu-skills](https://ossinsight.io/analyze/JimLiu/baoyu-skills) (10,873 stars), and [phuryn/pm-skills](https://ossinsight.io/analyze/phuryn/pm-skills) — a PM skills collection created March 1st that already has 8,029 stars — are doing numbers comparable to org-backed repos.

This is the dotfiles pattern playing out in real time. Developers want to know what *other developers they respect* are putting in their AGENTS.md. A personal skills repo is a window into someone's AI-augmented workflow. It's shareable in a way that raw config files never were.

## What This Looks Like to GitHub's Data

When I look at the trajectory, I see something that reminds me of when npm packages first started documenting themselves in README files. That sounds boring, but it was the moment the Node ecosystem stopped being esoteric and became accessible to everyone.

Skills are the documentation layer for AI-augmented development. They're how developers share *how they work* with agents, not just what they build.

The [OSSInsight analysis of everything-claude-code](https://ossinsight.io/analyze/affaan-m/everything-claude-code) shows a commit graph that doesn't look like a hype spike — the commit velocity has stayed elevated for 10 weeks. That's unusual. Most viral repos show a steep rise and then a 70-80% pullback in activity within a month. This one has sustained it.

## Where I'm Skeptical

Some of what's happening here might be inflated.

First, the "awesome list" problem. A significant chunk of these repos — `VoltAgent/awesome-openclaw-skills`, `sickn33/antigravity-awesome-skills` — are curated lists rather than original code. Stars on list repos are notoriously inflated relative to actual use. A developer stars a list to bookmark it; that's not the same engagement as starring a tool they're actively using.

Second, the skills format is still fractured. Claude Code uses `CLAUDE.md` and `.claude/skills/`. OpenCode has its own format. Codex reads `AGENTS.md`. Cursor does something different. The repos that are winning on stars now — particularly the aggregate collections — are betting on convergence that hasn't happened yet. If the agents land on incompatible formats, these collections either fragment or become unmaintainable.

Third, I can't tell from GitHub data alone whether people are *actually using* these skills or just starring them as interesting artifacts. [OSSInsight's repo analytics](https://ossinsight.io) gives us star/fork/commit data — but not how many developers have skills running in their day-to-day workflows.

There's also a deeper question that the data hints at but can't answer: most of these "skills" are not standalone. They rely on external data, persistent memory, or tool access to do anything useful. A prompt-based skill that says "query the database before answering" is only as good as the database connection it assumes. This suggests that skills alone may not be the full abstraction layer — they might be more like the visible tip of a much larger infrastructure stack that hasn't been named yet.

## A Prediction (Specific and Falsifiable)

By June 2026: at least one of the top-5 skills repos will have passed 150K stars, and there will be a publicly documented skills interoperability standard with signoff from at least two major agent projects.

Current trajectory suggests `affaan-m/everything-claude-code` is the most likely to hit 150K — it's adding ~2-3K stars/day even at 100K+, which is unusual. Most repos at that scale see growth slow significantly.

The interoperability standard is less certain. The incentives are weird — Claude Code and Codex might not want to make skills portable, because lock-in is valuable. But developer pressure is real, and the `vercel-labs/skills` tool is already trying to solve this with `npx skills` as a format-agnostic installer.

## The Two Things Worth Watching

If you want to track whether this is lasting signal or peak hype, watch two numbers on [OSSInsight](https://ossinsight.io):

1. **Fork velocity on `affaan-m/everything-claude-code`** — not stars. Forks mean customization, which means actual use. If fork/star ratio stays above 10%, that's genuine adoption.

2. **Commit activity on `openclaw/clawhub`** — the skills registry/directory play. Registry momentum is a lagging indicator, but it's the most honest signal of ecosystem health. High commit velocity in a registry means new skills are being published; that means developers are building things worth sharing.

At 10 weeks in, both indicators are still positive. This is either the fastest-growing developer workflow category I've tracked, or the most elaborate shared illusion. The data doesn't tell me which yet — but it's making me lean toward the former.
