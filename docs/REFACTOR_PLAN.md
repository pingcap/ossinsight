### 现有问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **单线程处理** | Ruby importer 单线程下载解析 | 速度慢，无法充分利用资源 |
| **技术栈老旧** | Ruby 2.7 + Rails 6 | 维护成本高，新人上手难 |
| **无增量更新** | 全量处理历史数据 | 重复计算，资源浪费 |
| **错误处理弱** | 失败后需手动重试 | 数据延迟，运维负担 |
| **无监控告警** | 缺少可观测性 | 问题发现滞后 |
| **数据质量** | 脏数据直接入库 | 影响分析准确性 |
| **扩展性差** | 垂直扩展为主 | 成本高效率低 |

## 🎯 重构目标

### 核心指标

| 指标 | 当前 | 目标 | 提升 |
|------|------|------|------|
| 数据处理延迟 | 24-48 小时 | <1 小时 | 24x+ |
| 吞吐量 | ~100 万条/小时 | 5000 万条/小时 | 50x |
| 错误恢复 | 手动 | 自动重试 + 断点续传 | - |
| 资源利用率 | <30% | >80% | 2.5x |
| 运维成本 | 高 | 低 | 自动化 |

### 功能需求

1. **实时数据管道** - GitHub Archive 每小时文件自动处理
2. **增量更新** - 仅处理新增/变更数据
3. **数据质量** - 自动清洗、验证、去重
4. **可观测性** - 完整监控、日志、告警
5. **弹性扩展** - 按需扩缩容，成本控制

## 🔧 技术选型

### 核心组件

| 组件 | 选型 | 理由 |
|------|------|------|
| **采集框架** | Python 3.11 + asyncio | 高性能异步 IO，生态丰富 |
| **JSON 解析** | orjson / ijson | 最快 JSON 库，流式支持 |
| **HTTP 客户端** | aiohttp + httpx | 异步 + 重试 + 连接池 |
| **消息队列** | Redpanda (可选) | Kafka 兼容，更轻量 |
| **流处理** | Apache Flink (可选) | 实时计算，状态管理 |
| **批处理** | DuckDB | 嵌入式 OLAP，极速 SQL |
| **任务调度** | Prefect / Airflow | 可视化 DAG，重试机制 |
| **监控** | Prometheus + Grafana | 标准方案，生态完善 |
| **日志** | Loki + Promtail | 轻量级，与 Prometheus 集成 |

### 运行第一个管道

```python
# pipelines/ingestion/gh_archive/downloader.py
import asyncio
import aiohttp

async def download_hour(hour_url):
    async with aiohttp.ClientSession() as session:
        async with session.get(hour_url) as resp:
            return await resp.read()

async def main():
    # 下载最近 1 小时数据
    url = "http://data.gharchive.org/2026-03-23-16.json.gz"
    data = await download_hour(url)
    print(f"Downloaded {len(data)} bytes")

asyncio.run(main())
```
